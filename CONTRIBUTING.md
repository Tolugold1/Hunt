# Contributing to Hunt

Thanks for your interest in contributing. Please read the [Contributing section in the README](README.md#contributing) for the full guide — it covers:

- Ground rules (all changes via PR, no direct pushes to `main`)
- How to fork, branch, commit, and open a pull request
- Branch naming conventions
- What makes a good PR vs what does not
- Code style expectations

**TL;DR:** fork → branch off `main` → make changes → `npx tsc --noEmit` passes → open a PR → wait for review. The maintainer performs all merges.
